## 이 페이지를 만든 이유

저는 기능을 구현하는 데서 멈추지 않고, 서비스가 왜 느려지고 왜 실패하는지 끝까지 이해하려고 노력하는 개발자입니다. 로그와 요청 흐름을 따라 문제를 작게 나누고, 개선 전후를 수치로 확인하며, 이해가 부족했던 기술은 별도의 학습 프로젝트로 다시 구현합니다.

이 페이지는 제가 사용해 본 기술을 단순히 나열하기 위한 공간이 아닙니다. 실제 프로젝트에서 어떤 문제를 발견했고, 어떤 판단과 구현으로 해결했으며, 그 과정에서 무엇을 배웠는지를 보여주는 저의 자기소개서입니다.

실무 경험은 공개 가능한 범위에서 문제·역할·결과를 중심으로 정리했습니다. 기관과 서비스를 식별할 수 있는 이름, 개인정보, 내부 주소와 상세 인프라 구성은 제외하고도 제가 문제를 해결한 과정과 기여가 명확하게 전달되도록 구성했습니다.

## 한눈에 보는 내용

| 경험 | 제가 수행한 내용 |
| --- | --- |
| 메인페이지 로딩 개선 | 최대 `3.5초`였던 로딩 시간을 `2.2초`로 줄여 약 `37%` 개선했습니다. |
| 영화 데이터 구축 | 영화 약 `13만 건`과 영화인 약 `7만 건`, 총 `20만 건` 규모의 데이터를 수집해 자체 DB를 구축했습니다. |
| 프로젝트 경험 | 데이터·성능·운영·배포 문제를 다룬 `4개` 프로젝트 경험을 문제와 결과 중심으로 정리했습니다. |
| 지속적인 학습 | 운영체제·알고리즘·HTTP 정기 스터디 `3개`와 업무 기반 개인 학습 프로젝트 `1개`를 이어가고 있습니다. |
| 공개 학습 프로젝트 | 실무의 궁금증을 일반화하고 합성 데이터로 다시 구현한 `6개` 저장소를 공개했습니다. |

## 기술 구성

저는 별도의 프레임워크나 외부 디자인 자산 없이 이 포트폴리오를 정적 웹 페이지로 구현했습니다.

- **Semantic HTML5**: 경력, 프로젝트, 학습 내용을 의미가 드러나는 문서 구조로 작성했습니다.
- **Responsive CSS**: CSS Grid와 Flexbox를 사용해 모바일부터 데스크톱까지 자연스럽게 읽히도록 구성했습니다.
- **Vanilla JavaScript**: 현재 섹션 표시, 헤더 상태, 점진적 노출 효과를 구현했습니다.
- **Accessibility**: 본문 바로가기, 키보드 포커스, 동작 줄이기 설정과 인쇄 스타일을 적용했습니다.
- **GitHub Actions · Pages**: `main` 브랜치의 검증된 정적 파일을 자동으로 배포합니다.
